# Security Notes

## 1) 계정 보안
- 공유 계정 사용 금지, 개인별 최소권한 계정 사용.
- 비밀번호 평문 저장 금지, 비밀관리 시스템 사용.
- 장기 미사용 계정 주기적 비활성화.

## 2) 세션/쿠키
- 세션 토큰, 쿠키를 로그/에러메시지에 출력 금지.
- 쿠키 파일/스토리지 상태 파일 커밋 금지.
- 세션 고정(Session Fixation) 방지를 위해 로그인 컨텍스트 분리.

## 3) 2FA
- 2FA 우회 자동화 시도 금지.
- 복구코드 저장 시 암호화 및 접근통제 필수.
- 운영 계정은 가능한 경우 하드웨어 기반 2FA 우선.

## 4) CAPTCHA
- CAPTCHA 우회 솔버 사용 금지.
- CAPTCHA 발생 시 수동 검증 또는 공식 API/파트너 경로 검토.

## 5) 약관(ToS)/법적 리스크
- 대상 플랫폼 약관 위반 가능성이 있는 자동화 금지.
- robots 정책/개발자 정책/브랜드 정책을 사전 검토.
- 데이터 수집/처리/보관 기간은 관련 법규(개인정보 포함) 준수.

## 6) 금지사항 요약
- 실제 사용자 자격증명 커밋 금지
- 민감 헤더/쿠키 외부 공유 금지
- 플랫폼 보안장치(2FA, CAPTCHA) 우회 로직 구현 금지
- 승인되지 않은 대량 호출/스크래핑 금지
